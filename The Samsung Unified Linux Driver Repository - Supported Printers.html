Supported Printer Models

The tables below indicate the minimum (lowest) and maximum (highest) versions of the driver that support each printer. If a maximum version is not specified, then everything equal to or above the minimum version supports the model. This also means that even if no "driver2" is indicated by the minimum version, it is still supported by driver2 unless the maximum version given indicates otherwise. If 2 minimum versions are present, the higher of the two reflects the last time an update to the driver likely affected support for the model (i.e., the lower minimum version may have bugs, or bugs may have been introduced later that were fixed in the second version given). For printers with 2 versions listed, there may have been additional updates between the listed versions.

Important caveats:
  • Printer models exclude the terminal letter codes (N, W, DN, etc.), because in general the differences do not matter for this purpose.
  • "Supported" simply means that the model exists in the ppd database or is configured for scanning; it is entirely possible that a particular "supported" version will still fail due to bugs in the driver, incompatibility with system libraries, or the current weather. Equally, in some cases "non-supported" models may actually work.
  • The min and max versions are based on the ones available in the repository. I do not package all versions of the driver, due to severe bugs or security issues with some releases, or simply no significant differences from packaged versions.
  • The scanning support column is a "-" if no scanner exists on the model, is blank if scanning support is unknown and/or unsupported, and has some text indicating which method(s) support the scanner. Samsung MFP requires installing the driver. Xerox MFP is standard libsane support, although you likely need either a fairly recent libsane (e.g., 1.0.22 or newer) or suld-driver-common-* installed (or maybe both).
  • While these tables are as accurate as I could make them within time constraints, there's a lot of guesswork involved.
If a particular driver version is listed as supporting your model but does not actually work, try the links to the questions at the top of this page. To install the appropriate version of the driver, select the suld-driver-* or suld-driver2-* package with the matching number (e.g., suld-driver-4.00.39 or suld-driver2-1.00.29).

The Samsung CJX-xxx printers are not supported by this driver, but many non-Samsung printers are (see the last table).
